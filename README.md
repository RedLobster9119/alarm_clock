# alarm_clock

目覚まし時計アプリ開発

＜要件定義＞
・時刻取得
・アラーム設定
・ユーザーに知らせる（サウンド、スクリーン）
・アラームを止める（ミニゲームでの勝利）
・電源を入れ直しても、時刻正常、アラーム設定変化なし

＜基本設計＞

各プロセス間をメッセージを用いて管理する。
アラーム設定時刻の情報については、共有メモリで管理する。

【入力 （全てメインファイルにメッセージを送る）】
・タッチパネル
・時刻（1秒毎にメッセージを送る）
・スイッチ

【出力 （全てメインファイルからメッセージを受信する）】
・スクリーン
・サウンド

【メイン】
・子プロセスの生成
・入力からのメッセージ受信
・出力へのメッセージ送信
・アラーム時刻の管理、メッセージ送信
